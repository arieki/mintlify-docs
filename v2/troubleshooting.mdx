# Troubleshooting Guide - v2.0

This guide helps you resolve common issues with Salsa as a Service v2.0, including AI forecasting and IoT sensor problems. If you can't find a solution here, our support team is ready to help.

## Quick Diagnostics

Before diving into specific issues, run these quick checks:

<Steps>
   <Step title="Check System Status">
      Visit our [status page](https://status.salsaasaservice.com) to see if there are any ongoing service issues.
   </Step>
   <Step title="Verify Internet Connection">
      Ensure you have a stable internet connection and can access other websites.
   </Step>
   <Step title="Clear Browser Cache">
      Clear your browser cache and cookies, then try logging in again.
   </Step>
   <Step title="Check Browser Compatibility">
      Ensure you're using a supported browser (Chrome, Firefox, Safari, Edge).
   </Step>
</Steps>

## Common Issues

### Login and Authentication

<AccordionGroup>
   <Accordion title="Unable to log into your account">
      1. **Verify Credentials**
         Double-check your email address and password. Ensure Caps Lock is off.

      2. **Reset Password**
         Use the "Forgot Password" link on the login page to reset your password.

      3. **Check Email**
         Look for account verification emails in your spam folder.

      4. **Contact Support**
         If the issue persists, contact support with your account email address.

      <Warning>
         **Security Note**: Never share your password with anyone. Our support team will never ask for your password.
      </Warning>
   </Accordion>
   <Accordion title="Issues with 2FA codes or setup">
      1. **Code Not Working**: Ensure your device's time is synchronized
      2. **Lost Device**: Use backup codes or contact support
      3. **App Issues**: Try a different authenticator app (Google Authenticator, Authy)
      4. **Setup Problems**: Follow our [2FA setup guide](/guides/two-factor-auth)
   </Accordion>
</AccordionGroup>

### Data Synchronization

<AccordionGroup>
   <Accordion title="POS Integration Issues: POS data not syncing with Salsa as a Service">
      <Tabs>
         <Tab title="Square">
            1. **Check Square Connection**
               Verify the Square integration is still connected in your dashboard.

            2. **Reauthorize Integration**
               Disconnect and reconnect your Square account.

            3. **Check API Permissions**
               Ensure Square API permissions include inventory management.
         </Tab>
         <Tab title="Toast">
            1. **Verify Toast API Key**
               Check that your Toast API credentials are valid and not expired.

            2. **Check Menu Mapping**
               Ensure salsa menu items are properly mapped to inventory items.
         </Tab>
         <Tab title="Clover">
            1. **Update Clover App**
               Ensure the Salsa as a Service app is updated in your Clover dashboard.

            2. **Check Permissions**
               Verify the app has necessary permissions for data access.
         </Tab>
         <Tab title="General">
            1. **Check Network Connection**
               Ensure stable internet connection for both systems.

            2. **Review Sync Logs**
               Check the integration logs for error messages.

            3. **Test Manual Sync**
               Try manually triggering a data sync from your dashboard.
         </Tab>
      </Tabs>
   </Accordion>
   <Accordion title="Inventory Sync Problems: Inventory levels not updating correctly">
      **Common Causes:**
         - Network connectivity issues
         - API rate limiting
         - Incorrect unit conversions
         - Data validation errors
      
      **Solutions:**

      1. **Check Sync Status**
         Navigate to **Settings > Integrations** and check the sync status.
      2. **Manual Sync**
         Try manually syncing inventory data from the integrations page.
      3. **Review Error Logs**
         Check the error logs for specific sync failures.
      4. **Contact Support**
         If issues persist, contact support with your sync logs.
   </Accordion>
</AccordionGroup>

### Production Issues

<AccordionGroup>
   <Accordion title="Quality Control Problems: Quality scores not recording correctly">
      <Steps>
         <Step title="Check Device Calibration">
            Ensure quality testing devices are properly calibrated
         </Step>
         <Step title="Review Testing Procedures">
            Verify staff are following correct testing protocols
         </Step>
         <Step title="Update Software">
            Ensure the mobile app is updated to the latest version
         </Step>
         <Step title="Check Permissions">
            Verify staff have proper permissions for quality testing
         </Step>
      </Steps>
   </Accordion>
   <Accordion title="Batch Tracking Issues: Production batches not tracking properly">
      <Note>
         **Batch Requirements**: Each batch must have a unique batch number and valid recipe assignment.
      </Note>

      **Troubleshooting Steps:**
      <Steps>
         <Step title="Verify Batch Data">
            Check that all required batch information is entered correctly
         </Step>
         <Step title="Check Recipe Assignment">
            Ensure the batch is assigned to a valid recipe
         </Step>
         <Step title="Review Staff Permissions">
            Verify the assigned chef has proper permissions for batch management
         </Step>
      </Steps>
   </Accordion>
</AccordionGroup>

### Mobile App Issues

<AccordionGroup>
   <Accordion title="App Crashes: Mobile app crashes or freezes">
      <Steps>
         <Step title="Update App">
            Ensure you're running the latest version of the Salsa as a Service mobile app.
         </Step>
         <Step title="Restart App">
            Force close the app and restart it.
         </Step>
         <Step title="Clear App Data">
            Clear app cache and data (you'll need to log in again).
         </Step>
         <Step title="Reinstall App">
            Uninstall and reinstall the app from your device's app store.
         </Step>
      </Steps>
   </Accordion>

   <Accordion title="Offline Mode Issues: App not working in offline mode">
      <Steps>
         <Step title="Check Offline Data">
            Ensure critical data is downloaded for offline use
         </Step>
         <Step title="Sync Before Going Offline">
            Manually sync data before losing internet connection
         </Step>
         <Step title="Check Storage Space">
            Ensure sufficient device storage for offline data
         </Step>
         <Step title="Update App">
            Use the latest app version with improved offline capabilities
         </Step>
      </Steps>
   </Accordion>
</AccordionGroup>

## Advanced Troubleshooting

### API Integration Issues

<AccordionGroup>
   <Accordion title="Authentication Errors: API requests returning 401 Unauthorized">
      <Steps>
         <Step title="Check API Key">Verify the API key is correct and not expired</Step>
         <Step title="Check Permissions">Ensure the API key has necessary permissions</Step>
         <Step title="Check Rate Limits">Verify you haven't exceeded rate limits</Step>
         <Step title="Check Request Format">Ensure requests are properly formatted</Step>
      </Steps>
   </Accordion>

   <Accordion title="Data Format Issues: API requests failing due to data format errors">
      <Note>
         **Common Issues:**
         - Incorrect date formats
         - Invalid data types
         - Missing required fields
         - Invalid JSON syntax
      </Note>

      <CodeGroup>
         ```javascript Solution
         // Example: Proper API request format
         const requestData = {
            name: "House Salsa",
            category: "salsa",
            batch_size: 5.0,  // Use numbers, not strings
            production_date: "2024-01-15T08:00:00Z",  // ISO 8601 format
            ingredients: [
            {
               ingredient_id: "ing_123",
               quantity: 2.0,
               unit: "lbs"
            }
            ]
         };
         ```
      </CodeGroup>
   </Accordion>
</AccordionGroup>
