# Salsa as a Service API v2.0

The v2.0 API adds GraphQL support, enhanced webhooks, and new AI forecasting endpoints.

## Base URL

<CodeGroup>
    ```text REST
    https://api.salsaasaservice.com/v2
    ```

    ```text GraphQL
    https://api.salsaasaservice.com/graphql
    ```
</CodeGroup>

<Note>
  <strong>New in v2.0:</strong> GraphQL support for flexible queries and subscriptions.
</Note>

## Authentication

All API requests require authentication using your API key.

<Note>
  <strong>API Key:</strong> Get your API key from the{" "}
  <a href="https://app.salsaasaservice.com/settings/api">Developer Settings</a> in your dashboard.
</Note>

### Authentication Header

<ParamField body="header" path="Authorization" type="string" required>
  The Authorization header expects a Bearer token. Use API key <code>Bearer YOUR_API_KEY</code>.
</ParamField>

## Rate Limits

- **Free Plan**: 1,000 requests per hour
- **Professional Plan**: 10,000 requests per hour  
- **Enterprise Plan**: 100,000 requests per hour

Rate limit headers are included in all responses:

<ParamField header="X-RateLimit-Limit" type="number">
  Your request limit for the current window. For example, <code>1000</code> for the Free Plan.
</ParamField>
<ParamField header="X-RateLimit-Remaining" type="number">
  Remaining requests in the current window. For example, <code>999</code> after one request on the Free Plan.
</ParamField>
<ParamField header="X-RateLimit-Reset" type="number">
  UNIX epoch timestamp when the window resets. For example, <code>1640995200</code>
</ParamField>

## Inventory Management 

### Get All Ingredients

Retrieve a list of all ingredients in your inventory.

```http
GET /ingredients
```

<CodeGroup>
```bash Request
curl --request GET \
  --url 'https://api.salsaasaservice.com/v2/ingredients' \
  --header 'Authorization: Bearer YOUR_API_KEY'
```
</CodeGroup>
<CodeGroup>
```json Response
  {
    "data": [
      {
        "id": "ing_123",
        "name": "Roma Tomatoes",
        "category": "vegetables",
        "current_stock": 25.5,
        "unit": "lbs",
        "supplier": "Fresh Farms Co.",
        "last_updated": "2024-01-15T10:30:00Z",
        "expiry_date": "2024-01-20T00:00:00Z",
        "quality_score": 8.5
      }
    ],
    "pagination": {
      "page": 1,
      "per_page": 50,
      "total": 1
    }
  }
```
</CodeGroup>

### Update Ingredient Stock

Update the stock level for a specific ingredient.

```http
PUT /ingredients/{ingredient_id}/stock
```
#### Path Parameters
<ParamField path="ingredient_id" type="string" required>
  The unique identifier of the ingredient to update.
</ParamField>

<CodeGroup>
```bash Request
  curl --request PUT \
  --url 'https://api.salsaasaservice.com/v2/ingredients/{ingredient_id}/stock' \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "quantity": 30.0,
    "unit": "lbs",
    "notes": "New shipment received"
  }'
```
</CodeGroup>
<CodeGroup>
```json Response
  {
    "id": "ing_123",
    "name": "Roma Tomatoes",
    "current_stock": 30.0,
    "unit": "lbs",
    "updated_at": "2024-01-15T11:00:00Z"
  }
```
</CodeGroup>

### Get Production History

Retrieve production history with filtering options.

```http
GET /production/batches
```
#### Query Parameters

<ParamField query="start_date" type="string">
  Filter batches starting from this date (inclusive). Format: ISO 8601.
</ParamField>
<ParamField query="end_date" type="string">
  Filter batches up to this date (inclusive). Format: ISO 8601.
</ParamField>
<ParamField query="recipe_id" type="string">
  Filter batches by recipe ID.
</ParamField>

<CodeGroup>
```bash Request
 curl --request GET \
  --url 'https://api.salsaasaservice.com/v2/production/batches?start_date=2024-01-01&end_date=2024-01-31&recipe_id=rec_456' \
  --header 'Authorization: Bearer YOUR_API_KEY'
```
</CodeGroup>

## Error Handling

The API uses standard HTTP status codes and returns detailed error information.

### Error Response Format

<ResponseField name="error.code" type="string" required>
  Machine-readable error code (for example, <code>INVALID_REQUEST</code>).
</ResponseField>
<ResponseField name="error.message" type="string" required>
  Human-readable explanation of what went wrong.
</ResponseField>
<ResponseField name="error.details" type="object">
  Optional, structured details (for example, validation errors).
</ResponseField>

<CodeGroup>
```json Response
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The request body is missing required fields",
    "details": {
      "missing_fields": ["name", "category"]
    }
  }
}
```
</CodeGroup>

### Common Error Codes


#### Client Errors

- `400 Bad Request` - Invalid request data
    - `401 Unauthorized` - Missing or invalid API key
    - `403 Forbidden` - Insufficient permissions
    - `404 Not Found` - Resource not found
    - `422 Unprocessable Entity` - Validation errors
    - `429 Too Many Requests` - Rate limit exceeded

#### Server Errors

- `500 Internal Server Error` - Server error
    - `502 Bad Gateway` - Upstream service error
    - `503 Service Unavailable` - Service temporarily unavailable

## Webhooks

Configure webhooks to receive real-time notifications about important events.

### Supported Events

<ParamField body="ingredient.low_stock">
  When ingredient stock falls below threshold
</ParamField>
<ParamField body="production.batch_completed">
  When a production batch is completed
</ParamField>
<ParamField body="quality.alert">
  When quality scores fall below acceptable levels
</ParamField>
<ParamField body="inventory.expiry_warning">
  When ingredients are approaching expiry
</ParamField>

### Webhook Configuration

```http
POST /webhooks
```

<CodeGroup>
```bash Request
 curl --request POST \
  --url 'https://api.salsaasaservice.com/v2/webhooks' \
  --header 'Content-Type: application/json' \
  --data '{
    "url": "https://your-app.com/webhooks/salsa",
    "events": ["ingredient.low_stock", "production.batch_completed"],
    "secret": "your_webhook_secret"
  }'
```
</CodeGroup>
